El examen final de tu curso de DevOps se centra en la **entrega de un proyecto completo**, no en un examen teórico tradicional. El alumno debe demostrar que domina tanto la parte técnica como la organizativa del flujo DevOps.  

### Contenido del examen / entregables
- **Repositorio Git**  
  El alumno debe entregar un repositorio bien estructurado, con ramas claras, commits descriptivos y un archivo README que explique el proyecto, su arquitectura y cómo desplegarlo.

- **Infraestructura como código**  
  Se espera un conjunto de scripts (Terraform, Ansible, etc.) que automaticen la creación de la infraestructura necesaria: redes, servidores, contenedores, balanceadores.

- **Pipeline CI/CD**  
  El alumno debe configurar un pipeline funcional (Jenkins, GitHub Actions, GitLab CI) que compile, ejecute pruebas, genere imágenes Docker y despliegue automáticamente en un entorno de staging o producción.

- **Contenedores y orquestación**  
  Se requiere un Dockerfile optimizado y un `docker-compose.yml` o manifiestos de Kubernetes que definan los servicios, volúmenes y redes.

- **Servidor web funcional**  
  El proyecto debe incluir un servicio web accesible (ejemplo: Nginx sirviendo una aplicación estática o dinámica), correctamente configurado y documentado.

- **Documentación técnica**  
  Manual de instalación, guía de despliegue y explicación de la arquitectura. Debe ser clara, práctica y permitir que cualquier persona pueda replicar el entorno.

- **Evidencias de pruebas**  
  Capturas de pantalla, logs o reportes que demuestren que el pipeline funciona, que los contenedores se levantan correctamente y que la aplicación responde.
